      전체 출결 관리

      학급 전체 출결을 캘린더와 필터 중심으로 확인하고, 추가·수정·삭제와 일괄 처리를 하는 곳입니다.

      출결 캘린더에서 날짜를 누르면 해당 날짜 기록 창이 열립니다. 주황 배경 날짜는 서류 미제출 기록이 있는 날입니다.

      학생별 출결

      개별 학생의 출결 기록, 서류 미제출, 삭제 이력, 유형 요약을 보는 곳입니다.

      서류 미제출 건수를 누르면 해당 학생의 미제출 목록을 확인하고 바로 제출 처리하거나 프린트할 수 있습니다.

      상담/누가기록

      날짜별 상담, 보호자 연락, 생활지도 기록을 출결 메모와 분리해 관리하는 곳입니다.

      후속 조치가 필요한 기록은 상태를 따로 표시해 나중에 다시 챙길 수 있습니다.

      서류 미제출 처리

      미제출 명단 프린트와 현재 목록 프린트는 현재 필터·정렬 조건을 반영하며, 출결 메모는 출력하지 않습니다.

      현재 필터 미제출 → 유형별 미인정 일괄 변경은 화면 필터에 걸린 서류 미제출 항목만 매핑에 따라 변경합니다.

      마지막 일괄 변경 되돌리기는 마지막 미인정 일괄 변경 직전의 출결 목록으로 되돌립니다.

      데이터 관리

      출결·상담 전체 복구용 백업은 암호화 저장을 사용하고, CSV/엑셀은 확인·제출·부분 이동용 자료로 사용하는 것을 권장합니다.

      업무/공지는 업무/공지 탭에서 별도 데이터로 관리합니다. 데이터 관리 창에서 두 백업 상태를 함께 확인하세요.

      암호화 저장 파일의 비밀번호는 복구할 수 없으므로 안전하게 보관하세요.

      배포와 데이터 이전

      HTML 파일만 옮기면 프로그램은 열리지만, 학생·출결·상담 데이터는 자동으로 따라가지 않습니다.

      기존 데이터를 옮길 때는 먼저 암호화 백업 파일을 내려받고, 새 PC에서 같은 HTML을 연 뒤 백업 불러오기를 사용하세요.

      학교망에서 엑셀 기능이 막히면 CSV 내보내기·불러오기를 사용하면 됩니다.

      화면 잠금과 저장 주의

      화면 잠금은 같은 브라우저에서 화면을 가리는 용도입니다. 자동 저장 데이터 자체를 완전히 암호화하는 기능은 아닙니다.

      자동 저장 데이터는 브라우저에 남습니다. 장기 보관과 복구는 암호화 백업 파일을 기준으로 사용하세요.

      설정

      설정 탭에서 학생 명단과 출석 유형을 관리할 수 있습니다.

      학생 명단을 등록하면 오늘 출결 입력 표와 학생별 출결 화면에 바로 반영됩니다.

      처음 사용 안내

      1 2 3 4

      먼저 할 일 세 가지

      1. 학생 명단

      HTML 파일을 최신 브라우저에서 열고, 설정 탭에서 학생 이름을 한 줄씩 붙여넣으면 빠른 입력 표가 만들어집니다.

      2. 빠른 출결

      오늘 출결 입력 탭에서 매일 결석·지각·조퇴·결과만 빠르게 체크합니다.

      3. 백업 저장

      자동 저장은 편의 기능이고, 안전한 보관은 암호화 백업 파일입니다.

      먼저 학생 명단을 등록하세요.

      반 학생 이름을 복사해서 한 줄에 한 명씩 붙여넣으면 됩니다.

      저장 기준

      브라우저 자동 저장

      같은 PC, 같은 브라우저에서 이어 쓰기 위한 편의 저장입니다.

      암호화 백업

      파일로 내려받아 따로 보관하는 안전 백업입니다. 비밀번호는 복구할 수 없습니다.

      내부 스냅샷

      최근 상태로 되돌리는 임시 복구용입니다. PC 고장 대비용은 아닙니다.

      Google Drive 또는 OneDrive의 학급일지백업 폴더를 처음 한 번 선택하면, 이후 백업 저장 버튼으로 해당 폴더에 바로 저장됩니다.

      매일 쓰는 순서

      1. 오늘 출결 입력 탭에서 오늘 날짜를 확인합니다.
      2. 결석·지각·조퇴·결과가 있는 학생만 선택하고 변경분 저장을 누릅니다.
      3. 서류 미제출은 학생별 출결 또는 전체 출결 관리에서 확인합니다.
      4. 하루가 끝나면 좌측의 출결·상담 백업 저장을 누릅니다. 백업 폴더가 지정되어 있으면 Google Drive/OneDrive 폴더에 바로 저장됩니다.

      데이터 관리

      출결·상담 데이터 관리
      오늘 백업 상태
      출결·상담: 확인 중
      업무/공지: 확인 중
      출결·상담 백업과 업무/공지 백업은 서로 다른 파일입니다. 둘 다 쓰는 경우 두 백업 상태를 함께 확인하세요.
      백업 위치 설정
      백업 폴더: 확인 중
      1. 처음 한 번만 Google Drive 또는 OneDrive 안의 학급일지백업 폴더를 지정합니다.
      2. 평소에는 좌측의 출결·상담 백업 저장만 누르면 지정 폴더에 바로 저장됩니다.
      3. 지정 폴더 저장을 지원하지 않는 PC에서는 자동으로 기존 다운로드 폴더 저장 방식으로 전환됩니다.
      권장 위치: Google Drive for desktop 또는 OneDrive의 학급일지백업 폴더. 다른 PC에서는 이 폴더의 최신 백업 파일을 불러오면 됩니다.
      고급: Firebase Cloud 동기화
      Firebase Cloud 동기화
      Firebase: 연결 안 됨
      Firebase Auth + Firestore

      같은 Firebase 계정으로 로그인한 PC와 스마트폰에서 일지 데이터를 저장/불러옵니다. Cloud 저장본은 암호화되며 최근 3개까지 선택해 복구할 수 있습니다.

      실행 환경: 확인 중
      Firebase SDK: 확인 중
      Firestore 규칙: 로그인한 본인 데이터만 접근
      사용 순서
      1. Firebase Authentication에서 이메일/비밀번호 계정을 만듭니다.
      2. Cloud login을 눌러 로그인합니다.
      3. 처음 기기에서 클라우드 저장을 누릅니다.
      4. 다른 기기에서 같은 계정으로 로그인한 뒤 클라우드 불러오기를 누르고 최근 3개 저장본 중 하나를 선택합니다.

      ※ 클라우드 불러오기는 현재 브라우저의 출결·상담·업무 데이터를 클라우드 저장본으로 교체합니다. 첫 사용 전 암호화 백업을 하나 받아두세요.

      테스트 사용 흐름
      1. Cloud login으로 Firebase 계정에 로그인합니다.
      2. PC에서 데이터를 입력한 뒤 클라우드 저장을 누릅니다.
      3. 스마트폰에서 같은 앱 주소와 계정으로 로그인한 뒤 클라우드 불러오기를 누릅니다.

      ※ GitHub Pages 같은 HTTPS 주소에서 사용하는 것을 권장합니다. file:// 직접 실행은 Firebase 로그인 흐름이 불안정할 수 있습니다.

      동기화 작업
      Firebase: 아직 로그인되지 않음
      개발자 설정: Firebase
      Firebase 설정은 HTML 코드에 포함되어 있습니다. 비밀번호가 아니며, 실제 접근 제한은 Firebase Authentication과 Firestore 규칙이 담당합니다.
      자주 쓰는 작업
      1. 평소 입력 중에는 브라우저 자동 저장을 편의 기능으로 사용합니다.
      2. 하루 업무가 끝나면 출결·상담은 반드시 암호화 백업 파일로 내려받습니다.
      3. 업무/공지를 사용했다면 업무/공지 탭의 암호화 백업도 따로 저장합니다.
